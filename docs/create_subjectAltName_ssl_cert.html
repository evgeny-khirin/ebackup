Introduction

This tutorial will show you have to generate a subjectAltName (SAN) Cert for Apache-2.2.x. This allows you to host multiple vhosts over SSL using a single IP address.

Edit openssl.cnf

/etc/pki/tls/openssl.cnf

Locate the following sections and uncomment them.

[ CA_default ]

# Extension copying option: use with caution.
copy_extensions = copy

[ req ]

req_extensions = v3_req
Notice that we are adding 5 different SANs. When we generate our CSR, all three names will be part of the CSR. If you want to add an additional name at a later date, you will need edit this section and generate a new CSR.

Locate the v3_req section and change it to:

[ v3_req ] # Extensions to add to a certificate request
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
subjectAltName = @alt_names
[alt_names]
DNS.1 = wikihub.berkeley.edu
DNS.2 = code.berkeley.edu
DNS.3 = beartracks.berkeley.edu
DNS.4 = ezsvn.berkeley.edu
DNS.5 = ezsvn-server.berkeley.edu

Generate Private Key

You will need to have the openssl tool installed. If you already have apache installed (hopefully you do) then you should be ok as this tool is bundled with apache. Verify that it is installed with:

which openssl

=> /usr/bin/openssl

Create 1024 bit RSA key, encrypted using Triple-DES and stored in a PEM format.

sudo openssl genrsa -des3 -out host.key 1024

Remove Private Key Passphrase

One side-effect of the pass-phrased private key is that Apache will ask for the pass-phrase each time the web server is started. This is not what we want. Luckily, mod_ssl includes the ability to use an external program in place of the built-in pass-phrase dialog, however, this is not necessarily the most secure option either. It is possible to remove the Triple-DES encryption from the key, thereby no longer needing to type in a pass-phrase. If the private key is no longer encrypted, it is critical that this file only be readable by the root user! If your system is ever compromised and a third party obtains your unencrypted private key, the corresponding certificate will need to be revoked.

Remove the pass-phrase from the key:

cp host.key host.key.org
openssl rsa -in host.key.org -out host.key

The newly created host.key file has no more passphrase in it.

Sign the CSR

At this point you will need to generate a self-signed certificate because you either don't plan on having your certificate signed by a CA, or you wish to test your new SSL implementation while the CA is signing your certificate. This temporary certificate will generate an error in the client browser to the effect that the signing certificate authority is unknown and not trusted.

Generate a temporary certificate which is good for 365 days:

openssl x509 -req -days 365 -in ca.csr -signkey ca.key -out ca.crt -extfile openssl.cnf -extensions v3_req

=> Signature ok

Install your Private Key and Cert

# Move the files to the correct locations
mv ezsvn.crt /etc/pki/tls/certs
mv ezsvn.key /etc/pki/tls/private/ezsv.key
mv ezsvn.csr /etc/pki/tls/private/ezsvn.csr
